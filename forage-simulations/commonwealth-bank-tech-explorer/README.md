# Commonwealth Bank Tech Explorer Simulation

## Overview

This simulation focused on exploring technology roles across software engineering, technology analysis, cybersecurity, and data science teams. The work involved stakeholder needs, user requirements, access groups, and data-based thinking for a banking application scenario.

## Skills Demonstrated

- Requirement gathering
- Stakeholder communication
- Access group identification
- Technology analysis
- Cybersecurity awareness
- Data-driven thinking
- Business and user needs analysis

## Tools & Concepts

- User requirements
- Stakeholder needs
- Access permissions
- Technology project planning
- Banking application concepts
- Demographic analysis
- Cross-functional technology teams

## What I Did

I practiced gathering and organizing stakeholder and user needs in a simulated technology environment. I identified appropriate groups for access to tools and reviewed how different technology functions support business goals.

The simulation also included analyzing demographic information to support advertising decisions for a new banking application. This helped me connect technology work with user needs, business goals, and data-supported decision making.

## Key Takeaways

This simulation helped me understand how cybersecurity and IT work often connect with broader technology teams. It reinforced the importance of communication, access planning, requirements gathering, and understanding how technical decisions support business outcomes.
